
## API Security Risk Analysis

### Objective

Perform a security assessment of a public API and identify potential security risks.

### Tools Used

* Postman
* Browser DevTools
* JSONPlaceholder API

### Security Checks Performed

* Authentication Review
* Authorization Review
* Data Exposure Analysis
* Rate Limiting Assessment
* Input Validation Testing

### Findings

1. Missing Authentication
2. Data Exposure Risks
3. No Visible Rate Limiting
4. Input Validation Weaknesses

### Outcome

Successfully analyzed a public API and documented identified security risks along with remediation recommendations.
